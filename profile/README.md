# overtura

작고 빠른 사이드 프로젝트를 병렬로 운영하는 개인 실험 조직입니다.

## 운영 원칙
- 1 thread = 1 PR
- 작은 scope 우선
- 코드 완벽함보다 사용자 flow 우선
- Codex-driven workflow
- 병렬 실험, 빠른 검증, 빠른 폐기

## 현재 운영 방식
- `.github` 에서 org 공통 템플릿과 기본 문서를 관리합니다.
- 새 프로젝트는 `template-web` 또는 `template-node`에서 생성합니다.
- 리뷰는 한국어 기준으로 진행합니다.
- 사람은 코드 디테일보다는 실제 동작 flow를 우선 확인합니다.

## 대표 레포
- `template-web`
- `template-node`
- 여기에 현재 대표 프로젝트 2~4개를 pin

## 이 org에서 기대하는 것
- 바로 실행 가능한 MVP
- 작은 PR
- 검증 루트가 적힌 README / PR
- 학습 포인트가 남는 결과물
