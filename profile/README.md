# Overtura

Small, parallel, Codex-driven side projects.  
Overtura is a personal experiment org for turning small ideas into runnable tools, templates, and workflow experiments.

## 🌱 What This Org Is For

- Ship small MVPs quickly
- Keep each experiment scoped and reviewable
- Prefer working user flows over perfect architecture
- Use Codex-assisted development as the default workflow
- Promote useful internal experiments into public repos when they are ready

## 📌 Public Repositories

| Repo | Type | What it is |
|---|---|---|
| [template-web](https://github.com/overtura/template-web) | Template | React/Vite web app starter for browser apps, design tools, and UI experiments |
| [template-node](https://github.com/overtura/template-node) | Template | TypeScript Node starter for CLIs, libraries, APIs, and ops tools |
| [repo-health-bot](https://github.com/overtura/repo-health-bot) | CLI | Repository health checks and self-improvement loop experiments |
| [action-ledger](https://github.com/overtura/action-ledger) | CLI | Markdown task scanning and Korean maintenance reports |
| [.github](https://github.com/overtura/.github) | Ops | Shared GitHub templates, profile README, and org defaults |

## 🧪 How Projects Usually Move

1. Start private and small.
2. Build the first runnable flow.
3. Keep README / PR notes focused on how to run and verify it.
4. Open source only the parts that are useful outside the experiment.

## 🤝 Working Style

- 1 thread = 1 PR
- Small scope first
- Korean reviews by default
- Fast validation, fast cleanup
- Clear verification notes over vague progress updates
